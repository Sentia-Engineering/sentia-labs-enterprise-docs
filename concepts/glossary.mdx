title: "Glossary"
description: "Key terms in Sentia, defined for research and design teams."
---

## A

**Agent**
A single synthetic respondent in a simulation. Each agent has demographic context, behavioral traits, memories, and a reasoning chain.

**Agent context**
Optional instruction in the simulation wizard that frames the agent's current task, such as "Right now you are evaluating a first-run onboarding flow."

## C

**Cohort**
The practical form of a population in the current app: a 10-100 agent audience created from a research brief, demographics, behavior patterns, traits, and optional artifacts.

**Confidence**
A report signal for how strongly supported the result appears, given the responses and cohort context.

**Convergence**
A live-run signal that shows whether additional agents are still changing the result distribution.

**Custom trait**
A workspace-specific audience dimension, such as AI literacy, security sensitivity, accessibility reliance, or cost scrutiny.

## D

**Deep mode**
The stronger simulation mode for higher-stakes studies and reports that need more robust evidence.

**Design Agent**
The Sentia workspace for importing Figma files, generating editable surfaces, creating variants, and handing selected designs to simulation.

**Design artifact**
A Design Agent surface attached to a simulation. It can include rendered HTML/CSS state and metadata from the design session.

## F

**Fast mode**
The lighter simulation mode for setup checks, exploration, and early design comparison.

**Figma source**
A connected Figma file, frame, page, or prototype URL used by Design Agent or the simulation wizard.

## P

**Population**
A reusable synthetic user audience. In the UI path, populations are built as focused 10-100 agent cohorts.

**Population lens**
The report section that explains how traits, demographics, or cohort slices reacted differently.

## R

**Research artifact**
A file uploaded to the Resource Library or attached during cohort creation: transcripts, surveys, persona docs, recordings, or experiment history.

**Resource Library**
The workspace area for uploaded research and saved simulation or comparison reports.

## S

**Simulation**
A synthetic user study against one scenario and one population.

**Simulation comparison**
A report comparing 2-3 completed simulations side by side.

**Social dynamics**
An experimental setting where later-wave agents can see sampled peer rationales from earlier waves.

## V

**Verbatim**
A representative quote-style response from an agent. When cohorts are research-grounded, voices should reflect language from the attached evidence.

## W

**Wave**
A batch of agents run in parallel as part of a simulation.

**Workspace**
An enterprise tenant. Data, users, integrations, populations, simulations, research artifacts, and reports are workspace-scoped.
