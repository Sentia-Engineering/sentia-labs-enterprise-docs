title: "Glossary"
description: "Key terms in Sentia — definitions in one place."
---

## A

**Adopter category**
A behavioral science classification — Innovator, Early Adopter, Early Majority, Late Majority, Laggard — that segments how users respond to new things. Sentia tags every agent and breaks results down by category by default.

**Agent**
A single synthetic respondent in a simulation. Each agent has a behavioral profile, retrieved memories, and runs a Memory → Reflection → Decision chain. Cost ≈ 3 LLM calls per agent.

**Agent narrative profile**
The structured persona attached to an agent: demographics, behavioral traits, usage history, preferences, concerns. Drives memory retrieval and decision reasoning.

## B

**Behavioral validation suite**
Five canonical behavioral economics experiments (dictator game, trust game, public goods, prisoner's dilemma, ultimatum) run against a sample of agents in a population to confirm fidelity. Pass rate ≥ 80% and fidelity score ≥ 0.7 are healthy.

## C

**Calibration**
The continuous process of comparing predicted vs. actual outcomes after a feature ships, and adjusting per-segment correction factors. The flywheel that makes Sentia more accurate every release.

**Capture**
A piece of input attached to a simulation: a URL screenshot, a localhost capture, a Figma frame, or a text description. Used by agents to "see" the change being evaluated.

**Confidence score**
How well-supported a simulation result is by the population's grounded data. 0–1 scale; > 0.8 is high confidence.

**Convergence**
A simulation has converged when the choice distribution stops changing wave-over-wave. Measured by Jensen-Shannon divergence falling below `convergenceThreshold` for two consecutive waves.

**Correction factor**
A per-segment multiplier applied to future simulation predictions, learned from past predicted-vs-actual gaps. Updates via Bayesian model after each shipped feature.

## D

**Decision**
A product change being evaluated. Lifecycle: `proposed → simulated → shipped → validated → reverted`. The unit of the calibration flywheel.

**Decision chain**
The three LLM calls each agent runs: Memory (recall), Reflection (reason), Decision (commit).

**Deep mode**
Simulation mode running 1,000–5,000 agents over 4–8 waves. Required for segment-level insights and high-confidence predictions.

**DPD (Demographic Parity Difference)**
A bias-detection metric. Sentia alerts when DPD exceeds a threshold across protected demographic segments in the population.

**Drift (prediction drift)**
A pattern where actual outcomes systematically diverge from predictions over time. Detected via CUSUM on the calibration accuracy series.

## F

**Fast mode**
Simulation mode running 200–500 agents over 2–3 waves. Good for quick directional signals; less reliable for segment-level claims.

**Fidelity score**
A 0–1 measure of how closely a population's behavior matches established behavioral economics priors. Reported on the validation page.

## G

**Ground truth**
The real-world outcome a simulation is calibrated against. Best source: A/B test results from Statsig or Eppo (weighted 3×). Acceptable: observational changes in Amplitude/Mixpanel/PostHog cohort metrics.

## I

**Interview grounding**
Embedding real customer verbatims (from Dovetail, Intercom, Zendesk) into agent memory so agents reflect real language and concerns instead of generic priors. Lifts accuracy from ~70% → ~85% normalized.

## J

**Jensen-Shannon divergence**
A symmetric, smoothed information-theoretic distance between two probability distributions. Sentia uses it to measure how much a wave changed the result. Lower = more stable.

## K

**KL divergence**
Kullback-Leibler divergence — an asymmetric distance between distributions. Used alongside J-S divergence for stability checks.

## N

**Normalized accuracy**
Accuracy adjusted for natural human behavioral inconsistency (≈81% test-retest baseline). A normalized accuracy of 1.0 means "as accurate as humans predicting their own behavior." Above 0.85 is excellent for decision-making.

**NPS prediction**
Per-segment and overall predicted change in Net Promoter Score from the simulated change. Surfaced on the Overview results tab.

## P

**Population**
A reusable, named, validated set of synthetic agents. Built from census filters + your analytics data + interview transcripts.

## S

**Segment**
A slice of the population by adopter category, plan tier, tenure, persona, or custom analytics cohort. All results are sliceable by segment.

**Ship detection**
Automatically detecting when a feature has rolled out, kicking off the 14-day measurement window. Sources: LaunchDarkly (flag > 50% rollout), GitHub PR merge with label, Slack `/shipped`, or manual.

## T

**Trigger**
A condition in a connected tool that auto-launches a simulation. Built-in: Linear/Jira status → "In Review", Figma frame named "simulate" or page "Testing", Slack `:simulate:` reaction.

## V

**Verbatim**
A direct quote from a real customer interview, support conversation, or NPS response. Embedded via Cohere, stored in Pinecone, retrieved by agents during the Memory step. The source of agents sounding like *your* users.

## W

**Wave**
A batch of agents run in parallel as part of one simulation. Convergence is measured between waves, not between individual agents.

**Workspace**
A tenant. All data, integrations, populations, and simulations are workspace-scoped. PostgreSQL row-level security enforces complete isolation.
