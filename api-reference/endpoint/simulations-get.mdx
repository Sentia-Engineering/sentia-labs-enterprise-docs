---
title: "Get simulation"
api: "GET /api/simulations/{simulationId}"
description: "Get current status, partial results, or final results of a simulation."
---

Returns the current state of a simulation. Use for polling, status display, or fetching final results.

## Request

```bash
curl -X GET https://api.sentialabs.ai/api/simulations/sim_8f9a2b... \
  -H "Authorization: Bearer $SENTIA_TOKEN" \
  -H "x-workspace-id: $WORKSPACE_ID"
```

## Path parameters

| Parameter | Type | Notes |
| --- | --- | --- |
| `simulationId` | UUID | The simulation to retrieve |

## Polling guidance

For real-time UI: poll every 5 seconds while `status` is `queued`, `running`, or `converging`. Stop polling on terminal states (`completed`, `failed`, `cancelled`).

Better: subscribe to webhooks instead of polling. See [Webhooks](/api-reference/webhooks).

## Fetching sub-results

The `results.*Url` fields point to result-specific endpoints. Each follows the same auth pattern. Some return paginated data:

| Result | Format | Pagination |
| --- | --- | --- |
| `segmentBreakdown` | JSON | Single response |
| `objections` | JSON | Single response |
| `agents` | JSON array | Cursor-paginated (default 100/page) |
| `transcripts` | JSON | Per-agent fetch |
| `cascade` | JSON | Single response |
| `calibration` | JSON | Updates over the 14-day window |

## Errors

| Status | Code | Cause |
| --- | --- | --- |
| 401 | `invalid_token` | Token missing or invalid |
| 403 | `workspace_access_denied` | Token doesn't grant access to this workspace |
| 404 | `resource_not_found` | Simulation ID doesn't exist or isn't in this workspace |
