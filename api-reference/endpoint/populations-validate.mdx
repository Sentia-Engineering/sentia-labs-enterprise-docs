---
title: "Validate population"
api: "POST /api/populations/{populationId}/validate"
description: "Run the 5-experiment behavioral suite against a sample of agents to verify behavioral fidelity."
---

Runs the canonical behavioral validation suite — dictator game, trust game, public goods, prisoner's dilemma, and ultimatum — against an available sample from the population. Returns when complete.

See [Validation (dashboard)](/populations/validation) for what the suite measures and how to read results.

## Request

```bash
curl -X POST https://api.sentialabs.ai/api/populations/pop_8f9a2b.../validate \
  -H "Authorization: Bearer $SENTIA_TOKEN" \
  -H "x-workspace-id: $WORKSPACE_ID"
```

## Path parameters

| Parameter | Type | Notes |
| --- | --- | --- |
| `populationId` | UUID | Must be in `ready` state |

## Outcome values

| Outcome | Meaning |
| --- | --- |
| `"passed"` | Pass rate ≥ 80% AND fidelity ≥ 0.7. Population is production-trustworthy. |
| `"marginal"` | Pass rate 60–80% OR fidelity 0.5–0.7. Use with caution. |
| `"failed"` | Either threshold below floor. Don't use for production simulations until fixed. |

## Errors

| Status | Code | Cause |
| --- | --- | --- |
| 404 | `resource_not_found` | Population doesn't exist |
| 409 | `resource_conflict` | Population is still building |
| 422 | `quota_exceeded` | Validation rate limit hit (rare) |
