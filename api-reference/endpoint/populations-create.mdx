api: "POST /api/populations"

---

Creates a new population. Generation runs asynchronously — the request returns immediately with status `building`.

## Request

```bash
curl -X POST https://api.sentialabs.ai/api/populations \
  -H "Authorization: Bearer $SENTIA_TOKEN" \
  -H "x-workspace-id: $WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "SMB trial users — May 2026",
    "description": "SMB users in their first 14 days of free trial",
    "agentCount": 1000,
    "region": "us",
    "filters": {
      "census": {
        "ageRange": [25, 54],
        "region": "US"
      },
      "cohorts": [
        { "provider": "amplitude", "cohortId": "1234567" }
      ]
    },
    "grounding": {
      "sources": [
        { "type": "dovetail", "projectIds": ["proj_abc"] },
        { "type": "intercom" }
      ],
      "verbatimDateRange": "P90D"
    }
  }'
```

## Body

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `name` | string | ✅ | Display name |
| `description` | string | Optional | Internal documentation |
| `agentCount` | int | ✅ | 1–100,000; 1,000 typical |
| `region` | enum | Optional | `"us"` or `"eu"`. Defaults to workspace region. |
| `filters.census` | object | Optional | Demographic filters (ageRange, income, region, etc.) |
| `filters.cohorts` | array | Optional | Analytics-cohort references |
| `grounding.sources` | array | Optional | Which qualitative sources to ground on |
| `grounding.verbatimDateRange` | ISO 8601 duration | Optional | Default `P90D` |
| `samplingStrategy` | enum | Optional | `"stratified"` (default) or `"random"` |

## Response — 201 Created

```json
{
  "id": "pop_8f9a2b...",
  "name": "SMB trial users — May 2026",
  "status": "building",
  "agentCount": 1000,
  "createdAt": "2026-05-25T14:32:17Z",
  "estimatedBuildDurationMs": 480000,
  "filters": { "...": "..." },
  "grounding": { "...": "..." }
}
```

After build completes, status transitions to `validating` → `ready` (or `failed`). Subscribe to `population.built` and `population.validated` webhooks to track.

## Status lifecycle

```
draft → building → validating → ready → (drifted) → archived
                              ↘ failed
```

See [Populations overview](/populations/overview).

## Errors

| Status | Code | Cause |
| --- | --- | --- |
| 422 | `validation_failed` | Filter or grounding misconfiguration |
| 422 | `unsupported_combination` | No analytics provider connected, or invalid cohort reference |
| 422 | `quota_exceeded` | Workspace population quota exceeded |
