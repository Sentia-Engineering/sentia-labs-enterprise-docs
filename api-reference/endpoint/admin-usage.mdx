---
title: "Usage analytics"
api: "GET /admin/usage"
description: "Programmatic access to workspace usage data for FinOps and budget tracking."
---

Returns per-period usage data for the workspace. Useful for piping into internal FinOps dashboards, billing systems, or alerting.

## Request

```bash
curl -X GET 'https://api.sentialabs.ai/admin/usage?startDate=2026-05-01&endDate=2026-05-31&granularity=day' \
  -H "Authorization: Bearer $SENTIA_TOKEN" \
  -H "x-workspace-id: $WORKSPACE_ID"
```

## Query parameters

| Parameter | Required | Notes |
| --- | --- | --- |
| `startDate` | ✅ | ISO date (`YYYY-MM-DD`) |
| `endDate` | ✅ | ISO date; inclusive |
| `granularity` | Optional | `day` (default), `week`, `month` |
| `breakdown` | Optional | Add a breakdown dimension: `user`, `population`, `mode`, `decisionTag`, `integration` |

## Granularity behavior

- `day` — one bucket per UTC day
- `week` — one bucket per ISO week (Monday start)
- `month` — one bucket per calendar month

Date ranges are clamped to billing cycle boundaries when relevant.

## Use cases

| Use case | Approach |
| --- | --- |
| FinOps dashboard | Daily granularity, `breakdown=user` |
| Budget alert pipeline | Daily granularity, monitor `burnRatePerDay` |
| Cost attribution to projects | `breakdown=decisionTag` (requires tagging discipline) |
| Identify high-consumption populations | `breakdown=population` |

## Errors

| Status | Code | Cause |
| --- | --- | --- |
| 400 | `invalid_date_range` | `startDate` > `endDate` or range exceeds 366 days |
| 403 | `role_insufficient` | Requires Owner role for full workspace usage |
