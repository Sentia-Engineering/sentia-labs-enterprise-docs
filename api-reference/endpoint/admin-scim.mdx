Sentia exposes a SCIM 2.0 endpoint **through WorkOS**. Your IdP (Okta, Entra ID, etc.) calls this endpoint to provision and deprovision users in real time.

You don't typically call SCIM endpoints directly â€” your IdP does. This page documents what your IdP sees, useful for debugging provisioning issues.

For end-user setup (configuring SCIM in Okta, Entra ID, etc.), see [SSO & SCIM](/admin/sso-scim).

## SCIM endpoint URL

Each workspace has a unique SCIM endpoint, provided by your CSM during setup. The general shape:

```
https://api.workos.com/scim/v2/<directory-id>/Users
https://api.workos.com/scim/v2/<directory-id>/Groups
```

This is what your IdP is configured to call. Sentia itself doesn't expose user-facing SCIM endpoints; WorkOS does, and Sentia consumes the WorkOS Directory Sync API server-side.

## Supported operations

| Operation | Notes |
| --- | --- |
| **GET /Users** | List users; supports `filter`, `startIndex`, `count` |
| **GET /Users/{id}** | Get specific user |
| **POST /Users** | Create user |
| **PATCH /Users/{id}** | Update user attributes |
| **PUT /Users/{id}** | Replace user |
| **DELETE /Users/{id}** | Deprovision user (immediate revocation in Sentia) |
| **GET /Groups** | List groups |
| **PATCH /Groups/{id}** | Add/remove members; updates Sentia role mapping |

## Attribute mapping

| SCIM attribute | Sentia field |
| --- | --- |
| `userName` | Email (must match domain authorized for workspace) |
| `emails[type=work].value` | Email |
| `name.givenName` | First name |
| `name.familyName` | Last name |
| `groups[].value` | Group memberships (mapped to roles) |
| `active` | `false` â†’ deprovision |

## Errors

SCIM uses standard HTTP status codes plus a SCIM-specific error schema:

```json
{
  "schemas": ["urn:ietf:params:scim:api:messages:2.0:Error"],
  "status": "409",
  "scimType": "uniqueness",
  "detail": "Email already exists"
}
```

| SCIM type | Cause |
| --- | --- |
| `uniqueness` | Email already in the directory |
| `mutability` | Tried to modify immutable attribute |
| `invalidValue` | Attribute value violates schema |
| `noTarget` | User/group ID not found |

## Audit logging

Every SCIM operation is audit-logged in Sentia with:

- Actor: `system:scim` (the IdP's WorkOS connection)
- Action: `member.scim_create` / `member.scim_update` / `member.scim_deprovision` / `member.role_change`
- Target: the affected user

Visible in Admin â†’ Audit log. See [Audit log](/admin/audit-log).
