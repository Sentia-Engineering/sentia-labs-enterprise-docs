The Sentia REST API is workspace-scoped, JWT-authenticated, and rate-limited. Use it to programmatically create simulations, manage populations, query results, administer workspaces, or wire Sentia into your internal tooling.

## Base URLs

| Region | URL |
| --- | --- |
| **Production (US)** | `https://api.sentialabs.ai` |
| **Production (EU)** | `https://api.eu.sentialabs.ai` |

Use the region matching your workspace residency. Requests to the wrong region's API will fail with a clear error message.

## Conventions

| | |
| --- | --- |
| **Auth** | `Authorization: Bearer <WorkOS-issued JWT>` |
| **Workspace** | `x-workspace-id: <uuid>` header on all data operations |
| **Content type** | `application/json` for request and response bodies |
| **Date format** | ISO 8601 with UTC timezone (`2026-05-25T14:32:17Z`) |
| **IDs** | UUID v4 unless noted |
| **Pagination** | `cursor`-based, with `next_cursor` in the response |
| **Errors** | Standard HTTP status codes + structured JSON error body |

## API stability

Sentia uses **resource-level versioning**, not a global version. Resources are versioned in their URL path when breaking changes occur (e.g., `/api/v2/simulations`). Non-breaking additions don't bump versions.

We commit to:

- **6-month deprecation window** for any resource version we plan to retire
- **Migration guides** for breaking changes, with side-by-side examples
- **No silent breaking changes** to existing resources

## API quotas

See [Rate limits](/api-reference/rate-limits). Defaults:

- 100 requests / minute / workspace (general)
- 10 simulation creates / minute / workspace
- 1,000 webhook deliveries / minute / provider
