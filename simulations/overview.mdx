A **simulation** is one run of N agents through one specific decision scenario. It produces a distribution of choices, segment breakdowns, an NPS prediction, top objections, and Sentia-generated recommendations.

## When to run a simulation

<CardGroup cols={2}>
  <Card title="Pre-launch validation" icon="rocket">
    Before shipping a non-trivial change — pricing, onboarding, plan structure, major UX flow.
  </Card>
  <Card title="Tradeoff resolution" icon="scale-balanced">
    Two competing designs and no consensus. Run both, compare segment receptivity.
  </Card>
  <Card title="Risk surfacing" icon="triangle-exclamation">
    "What could go wrong?" — Objections tab reveals concerns clustered from real verbatims.
  </Card>
  <Card title="Roadmap prioritization" icon="list-ol">
    Run candidate features in parallel against the same population. Compare predicted NPS lift, adoption curves.
  </Card>
</CardGroup>

Bad fits: bug fixes, copy tweaks, internal tooling. Agents simulate end users — they have no opinion on engineering churn.

## The four parts of a simulation

A simulation always has these four pieces:

```
┌────────────┐   ┌────────────┐   ┌────────────┐   ┌────────────┐
│  Decision  │   │  Scenario  │   │ Population │   │   Config   │
│            │   │ + captures │   │            │   │            │
└────────────┘   └────────────┘   └────────────┘   └────────────┘
```

| Part | What it is |
| --- | --- |
| **Decision** | The change being evaluated. Question, hypothesis, success criteria. |
| **Scenario + captures** | The "what does the change look like?" — Figma frames, captured UI, URL screenshots, or text. |
| **Population** | The "who is reacting?" — your validated population. |
| **Config** | The "how rigorous?" — mode (Fast/Deep), agent count, convergence threshold, max duration. |

You configure these in the 4-step [wizard](/simulations/wizard).

## How long it takes

| Mode | Agents | Duration | LLM cost (approx) |
| --- | --- | --- | --- |
| **Fast** | 200–500 | 2–5 min | ~600–1,500 calls |
| **Deep** | 1,000–5,000 | 8–25 min | ~3,000–15,000 calls |
| **Custom** | Up to 10,000 | Up to 60 min | Per agent × 3 calls |

Cost is metered per-token against your workspace contract allocation. Visible in Admin → Usage.
