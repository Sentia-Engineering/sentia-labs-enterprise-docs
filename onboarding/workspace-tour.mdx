Sentia is organized around the research and design workflow: bring in design work, define who should react to it, run a study, then preserve the evidence for the team.

## Main navigation

| Area | Shortcut | Use it for |
| --- | --- | --- |
| **Overview** | `O` | Workspace status, recent work, and high-level activity. |
| **Design Agent** | `D` | Import Figma files, generate editable surfaces, create variants, and send selected designs to simulation. |
| **Simulations** | `S` | Run synthetic user studies, monitor live runs, compare completed reports, and organize runs by project or tag. |
| **Populations** | `P` | Build behavioral cohorts of synthetic users from a natural-language audience brief and optional research artifacts. |
| **Resource library** | `R` | Upload research inputs and store simulation or comparison reports. |
| **Settings** | `⌘,` | Manage team access, Figma, design system guidance, integrations, and workspace settings. |

`⌘K` opens the command palette. `⌘L` opens Ask Sentia AI. `⌘⇧A` toggles the AI Co-Pilot panel.

## The first-week mental model

<Steps>
  <Step title="Connect Figma">
    Figma is the default design source in the current rollout. It powers both direct simulation inputs and Design Agent imports.
  </Step>
  <Step title="Add research context">
    Upload transcripts, survey CSVs, persona documents, call recordings, or prior experiment files in the Resource Library. These artifacts can ground future cohorts.
  </Step>
  <Step title="Build one cohort">
    Create a 10-100 agent behavioral cohort that represents the audience you want to study first.
  </Step>
  <Step title="Import or describe the design">
    Use Design Agent for Figma-driven variant work, or start a simulation directly with Figma frames or plain text.
  </Step>
  <Step title="Run and save the report">
    Watch live metrics, read the completed report, then save the PDF to the Resource Library for critique, planning, or stakeholder review.
  </Step>
</Steps>

## Roles

| Role | Good fit | Can do |
| --- | --- | --- |
| **Owner** | Workspace sponsor or operations lead | Full workspace control, billing, role management. |
| **Admin** | Research/design lead or platform owner | Manage members, Figma, design system docs, and advanced settings. |
| **Operator** | UX researcher, designer, or PM running studies | Build cohorts, run simulations, use Design Agent, and save reports. |
| **Viewer** | Stakeholders and read-only partners | Read reports, review runs, and follow workspace evidence. |

For most research and design teams, give active practitioners **Operator** access and reserve **Admin** for the people maintaining workspace configuration.

## What is not required on day one

You do not need analytics, calibration, or workflow automation to run the first study. Those are useful later for mature release loops, but current onboarding is Figma-first: connect designs, define a cohort, and run a clear study.
