The first time an Owner or Admin signs in, Sentia drops them into an **in-app setup guide** at `/onboarding`. It is a five-step wizard that gets a workspace from empty to a first real simulation. This page mirrors that guide so you can plan the rollout and bring the right people to each step.

Non-admin teammates skip the wizard entirely. They land on `/welcome`, see a workspace readiness summary (Figma · Population · First simulation · Calibration), and enter the app from there.

## Before the setup guide

Your Customer Success Manager finishes one thing before you sign in: **WorkOS AuthKit SSO** is provisioned against your IdP. There are no password sign-ins. If sign-in fails on the first attempt, see [Sign in with SSO](/onboarding/sso-setup).

## The in-app setup guide (5 steps)

The guide lives at `app.sentialabs.ai/onboarding` and auto-resumes at the first unfinished step. Each step has a **Skip** option except Welcome.

<Steps>
  <Step title="1. Welcome">
    Confirms the workspace name. No input required — click **Continue**.
  </Step>
  <Step title="2. Invite your team">
    Add teammates one at a time: email + role (**Admin**, **Operator**, or **Viewer** — Owner is reserved for the first sign-up and can be promoted from Settings → Team). Pending invitations show below the form. Invites use SSO, so the invitee's email must be authoritative under your IdP.
  </Step>
  <Step title="3. Connect your design tool">
    Paste a Figma personal access token (`figd_…`). The token needs `current_user:read`, `file_content:read`, and `file_comments:write`. Generate it at figma.com → Settings → Personal access tokens. The wizard auto-advances once Figma reports `connected`. You can skip and connect later from **Settings → Integrations**.
  </Step>
  <Step title="4. Build your first population">
    Clicking **Open population builder** takes you to `/populations/new`. Write a one-paragraph audience brief — Sentia infers behavior, psychology, contexts, and demographic filters from it live as you type. Pick a size (10–100 agents; default 50), optionally attach research artifacts, then create. The wizard auto-advances when the population reports done.
  </Step>
  <Step title="5. Launch your first simulation">
    Clicking **Create simulation** opens the 3-step simulation wizard at `/simulations/new`. Pick a scenario type, add context (Plain-text or Figma), describe the decision, pick the population you just built, set agent count, and **Launch**. Returning to the onboarding step after the run completes shows **Enter Sentia**.
  </Step>
</Steps>

After the last step, the dashboard transition runs and you land on `/overview`.

<Tip>
  You can leave the guide at any time via **Finish setup later** in the header. The next time an Owner or Admin signs in, the guide resumes at the first unfinished step.
</Tip>

## What everyone else sees

Operators, Viewers, Billing, and Guest roles land on `/welcome` with:

- Their role and a one-line description of what it grants.
- A workspace readiness checklist (Figma · Population · First simulation · Calibration) that reflects the live state of the workspace.
- An **Enter Sentia** button that takes them to `/overview`.

They never see the setup wizard.

## What is flag-gated in the current rollout

Several surfaces are behind feature flags during the current rollout. If a sidebar item or page is missing, the flag is off. Ask your CSM to enable any of these for your workspace:

| Surface | Flag |
| --- | --- |
| Design Agent (sidebar item, page, `D` shortcut) | `design-agent` |
| Simulations (sidebar item, page, `S` and `N` shortcuts) | `simulations-page` |
| Resource library (sidebar item, page, `R` shortcut) | `resource-library` |
| A/B comparison flow inside Simulations | `compare-feature` |
| Export PDF buttons on simulation reports and comparisons | `export-pdf-enabled` |
| Accessibility audit tab on simulation reports | `accessibility-audit` |

## After the wizard

You should have:

- SSO working, team invited (and pending invitations visible in Settings → Team).
- Figma connected (or explicitly skipped for now).
- One focused 10–100 agent population.
- One real simulation report.

From here the daily loop is: **design question → population → simulate → report → revise or decide.**
