## Two ways to add people

| Method | When to use | Provisioning speed |
| --- | --- | --- |
| **SCIM auto-provisioning** | Recommended for any workspace with > 10 users or strict IT requirements | Real-time (driven by IdP) |
| **Manual invite** | Small pilot teams, or before SCIM is set up | Immediate (email link) |

## Manual invite (Owners & Admins)

Manual invites are sent one at a time from the team tab. The same flow runs inside the onboarding wizard's "Invite your team" step.

<Steps>
  <Step title="Open Settings → Team">
    Press `⌘,` then click **Team**, or go to `app.sentialabs.ai/settings?tab=team`. From inside Settings → Team, `⌘I` opens the invite dialog directly.
  </Step>
  <Step title="Enter the invitee's email">
    Paste a single work email address. Bulk CSV paste is not supported in the dialog — repeat the flow for each invitee, or use SCIM for larger seeding.
  </Step>
  <Step title="Pick a role">
    Choose one of **Admin**, **Operator**, **Viewer**, **Billing**, or **Guest**. **Owner** is only available when the inviter is themselves an Owner. You can change roles later from the same tab.
  </Step>
  <Step title="Send">
    The invitee receives a magic link. They click → SSO with your IdP → land in the workspace. Pending invitations show in Settings → Team with **Resend** and **Revoke** actions.
  </Step>
</Steps>

<Note>
  Invites only work for email domains your IdP is authoritative for. Adding a user from a personal domain will fail at SSO. For external collaborators, use the **Guest** role.
</Note>

## Roles

Sentia has six workspace roles. Roles are enforced on every API request, and every change is audit-logged with actor, role, target, and timestamp.

| Role | Capabilities |
| --- | --- |
| **Owner** | Full control — billing, transfer ownership, delete workspace. Only an Owner can promote someone else to Owner. |
| **Admin** | User management, integrations, security, design system context. Cannot manage other Owners. |
| **Operator** | Runs simulations, manages populations and captures, uses Design Agent, saves reports. Most active researchers and designers land here. |
| **Viewer** | Read-only: simulation results and reports. Cannot run simulations or modify config. Good for execs and observers. |
| **Billing** | Invoices and payment methods only — no workspace data access. |
| **Guest** | Resource-scoped, time-bounded external access for partners or consultants. |

## Bulk seeding

For initial seeding of more than a handful of users, prefer **SCIM**. The in-app dialog accepts one email per invite and the team tab doesn't expose a CSV importer. SCIM also handles deprovisioning automatically.

## What teammates see on first sign-in

- **Owner / Admin** → the 5-step setup guide at `/onboarding` (Welcome → Invite team → Connect Figma → Build first population → Launch first simulation).
- **Operator / Viewer / Billing / Guest** → `/welcome` with a workspace readiness summary (Figma, Population, First simulation, Calibration) and an **Enter Sentia** button.

Returning users with onboarding completed land on `/overview`.
