Sentia ships continuously. This page summarizes customer-visible changes that affect onboarding and day-to-day workflows. For real-time deploy notifications, use the Slack Connect channel with your CSM.

For production service status, see [status.sentialabs.ai](https://status.sentialabs.ai).

## May 2026

### Figma-first onboarding

- The onboarding checklist now centers on Figma connection, cohort creation, first simulation, and saved reports.
- Figma setup uses a workspace personal access token in **Settings -> Integrations**.
- Required scopes are `current_user:read`, `file_content:read`, and `file_comments:write`.

### Design Agent

- Design Agent is now a primary dashboard area.
- Teams can import Figma files, index surfaces, generate editable variants, inspect changes, and post summaries back to Figma.
- Selected surfaces can be sent directly to simulation; each surface starts an independent run using the workspace default population.

### Populations and Resource Library

- Population creation is now a behavioral cohort studio with 10-100 agent cohorts.
- Resource Library includes **My research** for uploads and **Reports** for saved simulation or comparison evidence.
- Supported research uploads include CSV, TSV, DAT, TXT, PDF, DOCX, MP3, WAV, M4A, FLAC, OGG, MP4, MOV, and WEBM.

### Simulation reports

- Completed simulations now open as narrative reports with verdict, visuals, metrics, findings, concerns, recommendations, population lens, voices, and methodology.
- Comparison reports can compare 2-3 completed runs and be saved to the Resource Library.

## What changed in the docs

- Removed the old analytics-first quickstart from the primary onboarding path.
- Updated simulation setup from a 4-step model to the current 3-step wizard.
- Replaced older tab-based result guidance with the current report-reading workflow.
- Added Design Agent, Resource Library, design system settings, custom traits, simulation schedules, and social dynamics docs.
