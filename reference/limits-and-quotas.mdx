This page lists Sentia's platform limits. Many are configurable per workspace — talk to your CSM if defaults don't fit your usage.

## Rate limits

| Limit | Default | Notes |
| --- | --- | --- |
| **API requests (general)** | 100 / minute / workspace | Burst tolerance ~1.5× for short spikes |
| **Simulation requests** | 10 / minute / workspace | Separate from general rate limit |
| **Webhook deliveries (inbound)** | 1,000 / minute / provider | Per integration provider |
| **Auto-triggers** | 20 / day / workspace | Per-source sub-limits also apply |
| **Slack `:simulate:` reactions** | 10 / day / workspace | Sub-limit within auto-trigger budget |

API rate-limit responses return HTTP 429 with `Retry-After` header.

## Simulation limits

| Limit | Default | Maximum |
| --- | --- | --- |
| **Agents per simulation** | 1,000 | 10,000 |
| **Wave size** | 100–500 (mode-dependent) | n/a |
| **Convergence threshold** | 0.95 | 0.50 – 0.99 |
| **Max duration** | 30 min | 60 min |
| **Concurrent simulations** | 5 / workspace | Higher tiers available |
| **Captures per simulation** | 10 | 25 |

## Population limits

| Limit | Default | Maximum |
| --- | --- | --- |
| **Agents per population** | 1,000 | 100,000 |
| **Active populations** | Unlimited | n/a |
| **Validation sample size** | 200 agents | n/a |
| **Qualitative sources per population** | All connected | n/a |
| **Build duration** | 5–25 min typical | 60 min hard cap |

## Capture limits

| Limit | Default | Maximum |
| --- | --- | --- |
| **Uploaded image size** | 20 MB / file | 50 MB / file |
| **Files per capture set** | 10 | 25 |
| **URL capture pages** | 10 routes | 25 routes |
| **Capture retention** | 365 days | Custom per contract |

## Integration sync limits

| Limit | Default | Notes |
| --- | --- | --- |
| **Concurrent sync jobs** | 5 / workspace | Higher for enterprise tier |
| **On-demand sync** | 10 / hour / integration | Prevents abuse |
| **Webhook signature retries** | 3 attempts, exponential backoff | Failures audit-logged |

## Provider-specific limits

These are Sentia-side enforcement of provider rate limits to keep your account healthy:

| Provider | Sentia enforcement |
| --- | --- |
| **Anthropic** | Max 10 concurrent, 10 RPS, 20 burst |
| **Google Vertex** | Max 5 concurrent, 5 RPS, 10 burst |
| **OpenAI** | Max 5 concurrent, 5 RPS, 10 burst |
| **Figma** | Max 6 concurrent |
| **GitHub** | Max 6 concurrent |
| **Mixpanel** | Max 5 concurrent (rate-limited queue) |

If your team needs higher concurrency for high-volume workloads, your CSM can raise these in coordination with provider account teams.

## Storage quotas

| Resource | Default | Notes |
| --- | --- | --- |
| **Capture storage** | 100 GB / workspace | Auto-archive after 365 days |
| **Export bundles** | 7-day retention | Re-export anytime |
| **Audit log hot storage** | 90 days | Warm: 1 year. Cold: 7 years. |

## Seat & contract limits

These come from your contract:

| Resource | Set by |
| --- | --- |
| **Active seats** | Annual contract |
| **Simulation budget (LLM calls)** | Annual contract |
| **Connected integrations** | Most contracts: unlimited |
| **Workspaces** | Annual contract; typically 1 |

See [Billing & usage](/admin/billing-and-usage).

## Custom limits

Workspaces with high-volume usage can request raised limits:

- Simulation concurrency
- Provider rate limits
- Storage retention
- Webhook delivery rates
- API request rates

Submit requests through your CSM. Most can be raised within hours; some require coordination with infrastructure or provider teams.

## What happens when you hit a limit

| Limit type | Behavior |
| --- | --- |
| **API rate limit** | HTTP 429 + `Retry-After` header. Client should back off. |
| **Simulation concurrency** | New requests queue automatically; visible in the Simulations list as `queued` |
| **Auto-trigger rate limit** | Trigger fires but simulation isn't queued; logged for visibility |
| **Storage quota** | Captures fail to upload until cleanup or quota raise |
| **Contract allocation** | Per workspace alert config (warning at 75%, hard cap at 100% by default) |
